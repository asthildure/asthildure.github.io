Veiledning for webhosting på GitHub Pages

GitHub Pages er en gratis tjeneste fra GitHub som lar deg hoste statiske nettsider direkte fra et GitHub-repositorium. Denne veiledningen vil vise deg hvordan du enkelt kan sette opp og hoste din egen nettside på GitHub Pages.

1. Opprett et GitHub-repositorium

Registrer deg på GitHub: Hvis du ikke allerede har en konto, opprett en på github.com.

Opprett et nytt repositorium: Klikk på "New repository" og gi det et passende navn. For en personlig eller organisasjonsside, bruk dittbrukernavn.github.io.

2. Forbered nettsiden

Lag din statiske nettside: Dette kan være HTML, CSS, og JavaScript-filer. GitHub Pages støtter ikke server-side språk som PHP.

Test nettsiden lokalt: Sørg for at alt fungerer som det skal på din lokale maskin.

3. Last opp nettsiden til GitHub

Klon repositoriet: Bruk Git til å klone repositoriet til din lokale datamaskin.

Legg til filene: Flytt nettstedets filer til repositoriet.

Commit og push: Bruk Git til å lagre (commit) og laste opp (push) filene til GitHub-repositoriet.

4. Aktiver GitHub Pages

Gå til repositoriets innstillinger: Klikk på "Settings" i ditt repositorium.

Aktiver GitHub Pages: I seksjonen "GitHub Pages", velg branchen du vil bruke (vanligvis main), og klikk på "Save".

5. Tilpasse og oppdatere nettsiden

Tilpass domenet: Du kan koble til ditt eget domenenavn hvis ønskelig.

Oppdatering: For å oppdatere nettstedet, last opp nye filer til repositoriet og commit/push endringene.

Gratulerer! Du har nå en nettside hostet på GitHub Pages. Besøk {dittbrukernavn}.github.io for å se din nettside live.
